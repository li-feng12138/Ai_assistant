小航 - 校园信息AI助手 项目说明书

一、 项目概述
项目名称：小航 - 校园信息AI助手 
开发语言：Python 
运行方式：streamlit run day6_new.py 
目标用户：郑州航空工业管理学院的新生、在校生、教师 
核心定位：基于校园资料文档，为不同身份用户提供准确的校园信息查询AI问答服务



二、 功能模块
1. 多身份AI问答
 支持 新生 / 在校生 / 教师 三种身份
 不同身份对应不同的角色语气提示词（亲切学长、同学朋友、正式专业）
 身份动态分类推荐提问：
   新生：3个分类按钮（🎒 新生入学、🏫 校园生活、🚌 校园交通），每个分类下4个问题
   在校生：2个分类按钮（🏫 校园生活、🚌 校园交通），每个分类下4个问题
   教师：4个平铺按钮（院系电话、教务财务、紧急处理、心理咨询）
2. 多会话管理
 支持 新建、切换、重命名、删除 会话
 会话数据通过 JSON 文件持久化存储在 sessions/ 目录
 文件命名格式：conv_{会话名}.json
 重命名时进行 空值校验、同名校验，防止冲突
3. 知识库管理
 自动读取 Data/ 目录下 [0-9]*.md 格式的校园资料文件（共5份）：
   01新生入学.md
   02办事流程.md
   03电话黄页.md
   04应急防骗.md
   05交通出行.md
 资料内容作为AI回答的唯一参考来源，新增文档无需修改代码
4. 防幻觉安全机制（6条硬性规则）
 禁止编造任何具体数据（电话、金额、日期等）
 金钱话题必须附加反诈提醒
 紧急问题优先提供心理援助热线和保卫处电话
 资料无相关内容时如实告知，禁止猜测
 不支持查询个人个人信息（成绩、宿舍号等）
 每次回答末尾标注资料来源
5. 流式输出与加载动画
 调用API时启用 stream: True，逐字显示AI回答
 使用 st.empty() 占位组件实现打字机效果
 API调用期间显示 st.spinner 加载动画："AI正在整理校园相关答案，请稍候..."
6. 性能统计
 每条AI回答底部显示接口耗时（秒，保留2位小数）和回复字数
 统计信息持久化到会话JSON中，刷新页面后仍可见
7. 全屏电话黄页
 侧边栏点击按钮触发全屏弹出页面，右上角关闭按钮返回
 涵盖：行政服务、院系联系、生活服务、安全应急、心理关怀五大类电话
 采用 st.stop() 拦截后续渲染，实现页面切换效果

四、 核心流程
graph TD
    A["用户打开页面"] --> B{"是否有会话?"}
    B -- 否 --> C["自动创建默认会话"]
    B -- 是 --> D["加载历史会话"]
    C --> E["选择身份"]
    D --> E
    E --> F["显示分类推荐按钮"]
    F --> G["点击推荐问题 / 输入框输入"]
    G --> H["handle_question 存入 pending_q 并 rerun"]
    H --> I["历史消息渲染"]
    I --> J["检测 pending_q，显示 spinner 加载动画"]
    J --> K["读取md知识库 → 组装Prompt → 流式调用API"]
    K --> L["逐字显示AI回答 + 性能统计"]
    L --> M["保存会话到sessions目录 → rerun"]
