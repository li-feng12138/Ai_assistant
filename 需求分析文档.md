# 郑州航院校园AI问答助手"小航" —— 需求分析文档

## 一、项目概述
### 1.1 项目名称
郑州航空工业管理学院校园AI问答助手 —— "小航"

### 1.2 项目背景
新生入学及日常校园生活中，学生、教师经常需要了解报到流程、办事流程、校园联系方式、应急防骗、交通出行等信息。
传统方式依赖人工咨询，效率低、响应慢。本项目开发一款基于大语言模型的校园AI问答Web助手，基于校园资料提供智能问答服务。

### 1.3 目标用户
| 用户角色 | 描述 |
| 新生 | 刚入学，对校园不熟悉，需要报到、缴费、宿舍、交通等指引 |
| 在校生 | 需要办理在读证明、校园卡补办、了解校园生活和交通等事务 |
| 教师 | 需要了解校园行政流程、联系方式、应急处理等信息 |

## 二、功能需求
### 2.1 功能总览
| 编号 | 功能模块 | 描述 |
| F01 | 身份选择 | 下拉框选择新生/在校生/教师身份，系统据此提供针对性回答 |
| F02 | 智能问答 | 基于校园知识库资料，通过AI流式输出回答用户问题 |
| F03 | 分类推荐提问 | 按身份动态展示分类按钮，点击即可快速提问 |
| F04 | 电话黄页 | 侧边栏按钮触发全屏弹出页面，查看校园常用电话 |
| F05 | 多会话管理 | 支持新建、切换、重命名、删除多个独立对话 |
| F06 | 加载动画 | API调用期间显示spinner加载提示 |
| F07 | 性能统计 | 每条AI回答底部显示接口耗时和回复字数 |

### 2.2 功能详细描述
#### F01 身份选择
- 页面顶部下拉框选择身份：新生、在校生、教师
- 选择后自动保存至当前会话，支持随时切换
- 不同身份展示不同的推荐提问分类

#### F02 智能问答
- 底部输入框输入自然语言问题，按Enter发送
- 流式输出（SSE），逐字显示AI回答，带光标动画
- 回答严格基于校园知识库（5份md文档），不编造信息
- 每条回答末尾标注参考来源（`📎 资料来源：《xxx》`）
- 超出知识范围的问题，如实告知并建议咨询相关部门
- 每条AI回答底部以浅灰色小字显示接口耗时和回复字数

#### F03 分类推荐提问
按身份动态展示分类按钮，点击分类后展示对应快捷问题：
- **新生**（3个分类）：🎒 新生入学、🏫 校园生活、🚌 校园交通
- **在校生**（2个分类）：🏫 校园生活、🚌 校园交通
- **教师**（平铺4个按钮）：院系电话、教务财务联系、紧急处理、心理咨询
- 每个分类下预设4个高频问题，点击按钮直接发送

#### F04 电话黄页
- 侧边栏点击"📞 校园电话黄页"按钮触发全屏弹出页面
- 右上角"✖ 关闭"按钮返回主界面
- 涵盖：行政服务、院系联系、生活服务、安全应急、心理关怀五大类电话
- 采用表格分栏布局展示

#### F05 多会话管理
- 左侧边栏展示会话列表，支持新建/切换/重命名/删除
- 每个会话独立保存身份选择和对话记录
- 会话数据持久化存储为JSON文件（`sessions/` 目录）
- 重命名时进行空值校验、同名校验，防止冲突
- 页面刷新后自动恢复历史会话

#### F06 加载动画
- 用户提交提问后，API调用期间显示 `st.spinner` 转圈加载
- 提示文字："AI正在整理校园相关答案，请稍候..."
- API返回后动画自动消失

#### F07 性能统计
- 发送请求前记录起始时间，API返回后计算耗时（保留2位小数，单位秒）
- 自动统计AI回复总字符数（含标点不含空白换行）
- 以浅灰色小字展示在每条AI回答底部，持久化到会话数据中

## 三、安全与边界规则
| 编号 | 规则名称 | 描述 |
| R01 | 数据源边界 | 仅依据校园资料回答，资料中无相关信息时如实告知并建议咨询相关部门 |
| R02 | 真实信息边界 | 禁止编造电话、地址、金额、时间等具体信息，所有数据必须来自校园资料 |
| R03 | 资金反诈边界 | 涉及转账/缴费时，末尾附加反诈温馨提示 |
| R04 | 心理应急边界 | 涉及心理危机/人身安全时，优先提供心理援助热线（400-161-9995）和保卫处电话（0371-63456301） |
| R05 | 权限边界 | 拒绝查询成绩、课表、余额等个人数据，引导至相关部门 |
| R06 | 溯源边界 | 每条回答末尾标注 `📎 资料来源：《xxx》` |

## 四、知识库设计
知识库文件存放在 `Data/` 目录下，代码通过 `glob("[0-9]*.md")` 自动加载所有编号开头的md文件，新增文档无需修改代码。
| 文件 | 内容 | 覆盖场景 |
| 01新生入学.md | 报到流程、缴费说明、宿舍安排、防骗提醒 | 新生入学相关 |
| 02办事流程.md | 在读证明、校园卡补办、转专业、图书馆、请假 | 日常办事相关 |
| 03电话黄页.md | 行政/院系/生活/应急电话 | 电话查询相关 |
| 04应急防骗.md | 常见骗局、反诈提醒、心理援助、紧急处理 | 安全应急相关 |
| 05交通出行.md | 地铁/公交/共享单车/从主要站点到校路线 | 校园交通相关 |

### 5.2 技术选型
| 项目 | 选型 | 说明 |
| 编程语言 | Python | 简洁易维护 |

| HTTP库 | requests | 发送API请求，支持SSE流式读取 |
| 知识库格式 | Markdown | 易编辑、易维护 |

| 运行方式 | Streamlit Web应用 | 浏览器访问 |

### 5.3 核心模块说明
| 函数 | 职责 |
| `load_all_docs()` | 读取Data目录下所有编号md文件，合并返回全部文本 |
| `get_identity_prompt(identity)` | 根据用户身份返回对应的角色语气提示词 |
| `get_safety_rules()` | 返回6条防幻觉硬性规则 |
| `build_system_prompt(identity, campus_docs)` | 拼接身份提示词 + 校园资料 + 安全规则 |
| `chat_with_ai_stream(user_question, system_prompt, placeholder)` | 流式调用API，逐字显示回答，含异常处理 |
| `save_conversations()` | 将所有会话保存到sessions目录（JSON文件） |
| `load_conversations()` | 从sessions目录加载所有会话文件 |
| `delete_conversation(conv_name)` | 删除指定会话的JSON文件 |

## 八、总结
本项目实现了一个基于DeepSeek大模型的校园AI问答Web助手，具备身份感知、知识库问答、分类推荐提问、全屏电话黄页、多会话管理、流式输出、加载动画、性能统计等功能，并通过六条安全边界规则确保回答的可靠性和安全性。
