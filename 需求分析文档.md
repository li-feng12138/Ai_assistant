# 郑州航院校园AI问答助手"小航" —— 需求分析文档

## 一、项目概述

### 1.1 项目名称
郑州航空工业管理学院校园AI问答助手 —— "小航"

### 1.2 项目背景
新生入学及日常校园生活中，学生、教师经常需要了解报到流程、办事流程、校园联系方式、应急防骗等信息。
传统方式依赖人工咨询，效率低、响应慢。本项目开发一款基于大语言模型的校园AI问答助手，基于校园资料提供智能问答服务。

### 1.3 目标用户
| 用户角色 | 描述 |
|---------|------|
| 新生 | 刚入学，对校园不熟悉，需要报到、缴费、宿舍等指引 |
| 老生 | 需要办理在读证明、校园卡补办、转专业等事务 |
| 教师 | 需要办理科研经费报销、调课、设备报修等行政事务 |

## 二、功能需求

### 2.1 功能总览
| 编号 | 功能模块 | 描述 |
|------|---------|------|
| F01 | 身份选择 | 用户选择新生/老生/教师身份，系统据此提供针对性回答 |
| F02 | 智能问答 | 基于校园知识库资料，通过AI回答用户问题 |
| F03 | 快捷提问 | 每种身份预设4个常见问题，输入编号即可快速提问 |
| F04 | 电话黄页 | 离线查看校园常用电话，无需联网 |
| F05 | 别名识别 | 自动将"信院""教务"等简称转为全称，提升理解能力 |

### 2.2 功能详细描述

#### F01 身份选择
- 主菜单提供"选择身份并开始提问"入口
- 展示三种身份：新生、老生、教师，附带身份描述
- 支持随时切换身份（输入 `0` 返回身份选择）
- 选择后进入对应身份的对话模式

#### F02 智能问答
- 用户输入自然语言问题，系统调用AI生成回答
- 回答严格基于校园知识库（md文档），不编造信息
- 每条回答末尾标注参考来源（如 `📎 来源：01新生入学.md`）
- 超出知识范围的问题，统一引导至校值班室（0371-63456789）
- 输入 `q` 退出对话，输入 `0` 切换身份

#### F03 快捷提问
每种身份预设4个高频问题：
- **新生**：报到材料、缴费方式、宿舍安排、常见骗局
- **老生**：在读证明、校园卡补办、转专业、图书馆时间
- **教师**：科研经费报销、调课申请、设备报修、项目申报
- 输入编号（1-4）直接发送对应问题

#### F04 电话黄页
- 主菜单选择 `2` 查看校园电话黄页
- 离线可用，直接读取本地 `03电话黄页.md` 文件
- 涵盖：行政服务、院系联系、生活服务、安全应急四大类电话

#### F05 别名识别
- 支持校园常用简称自动转换：信院→信息工程学院、航院→航空工程学院、教务→教务处 等共10组
- 在用户输入送入AI前自动完成替换

## 三、安全与边界规则

| 编号 | 规则名称 | 描述 |
|------|---------|------|
| R01 | 数据源边界 | 仅依据校园资料回答，无关问题统一引导至校值班室 |
| R02 | 真实信息边界 | 禁止编造电话、地址、金额、时间等具体信息 |
| R03 | 资金反诈边界 | 涉及转账/缴费时，末尾附加反诈温馨提示 |
| R04 | 心理应急边界 | 识别到轻生/自残等关键词时，立即输出心理援助热线（0371-63456303 / 400-161-9995） |
| R05 | 权限边界 | 拒绝查询成绩、课表、余额等个人数据，引导至官方系统 |
| R06 | 溯源边界 | 每条回答末尾标注 `📎 来源：xxx.md` |

## 四、知识库设计

| 文件 | 内容 | 覆盖场景 |
|------|------|---------|
| 01新生入学.md | 报到流程、缴费说明、宿舍安排、防骗提醒 | 新生入学相关 |
| 02办事流程.md | 在读证明、校园卡补办、转专业、图书馆、请假 | 日常办事相关 |
| 03电话黄页.md | 行政/院系/生活/应急电话 | 电话查询（离线） |
| 04应急防骗.md | 常见骗局、反诈提醒、心理援助、紧急处理 | 安全应急相关 |

## 五、技术方案

### 5.1 技术架构
用户终端（命令行） 
 业务逻辑层（day5.py）
 身份管理模块（IDENTITIES）
 别名解析模块（resolve_alias） 
 知识库加载模块（load_md） 
 AI调用模块（call_api） 
 硅基流动API（DeepSeek-V3）

### 5.2 技术选型
| 项目 | 选型 | 说明 |
|------|------|------|
| 编程语言 | Python | 简洁易维护 |
| AI模型 | DeepSeek-V3 | 通过硅基流动API调用 |
| API地址 | https://api.siliconflow.cn/v1/chat/completions | 硅基流动平台 |
| HTTP库 | requests | 发送API请求 |
| 知识库格式 | Markdown | 易编辑、易维护 |
| 运行方式 | 命令行交互 | 无需额外UI依赖 |

### 5.3 核心模块说明
| 函数 | 职责 |
|------|------|
| `load_md(filename)` | 加载指定md知识库文件 |
| `resolve_alias(text)` | 将用户输入中的校园简称替换为全称 |
| `call_api(messages)` | 调用硅基流动API，含超时/网络异常处理 |
| `ai_answer(user_input, identity_desc, source_files)` | 组装prompt并调用AI获取回答 |
| `chat_loop(identity_key)` | 对话循环，处理快捷提问和自由输入 |
| `main()` | 主程序入口，控制主菜单流程 |

## 六、运行环境

- **操作系统**：Windows
- **Python版本**：3.x
- **依赖库**：`requests`
- **网络要求**：智能问答需联网；电话黄页可离线使用
- **启动方式**：`python day5.py`
