# 小航 - 校园信息AI助手 项目说明文档

## 一、项目概述
本项目是一个基于Streamlit搭建的校园信息查询Web应用，接入硅基流动大模型API（DeepSeek-V3），结合本地校园知识库文档，为郑州航空工业管理学院的师生提供准确的校园信息查询服务。
项目采用身份驱动的交互设计，根据用户身份（新生/在校生/教师）动态调整回答风格和推荐问题，同时内置防幻觉机制，确保AI回答严格基于校园资料，不编造数据。
## 二、技术栈
- 前端界面：Streamlit（Python Web框架）
- 大模型服务：硅基流动API（https://api.siliconflow.cn）
- 模型：DeepSeek-V3（deepseek-ai/DeepSeek-V3）
- 调用方式：requests库直接发送HTTP流式请求
- 数据存储：JSON文件（会话持久化）+ Markdown文件（校园知识库）

## 四、核心功能模块
### 4.1 提示词管理（第30-99行）
负责构建发送给大模型的系统提示词，由三个函数分层协作：
- get_identity_prompt(identity)：根据用户身份返回对应的角色语气设定。新生对应亲切学长语气，在校生对应轻松同学语气，教师对应专业助手语气。
- get_safety_rules()：返回6条防幻觉硬性规则，包括禁止编造数据、反诈提醒、紧急问题热线优先、无资料如实告知、拒绝查个人信息、标注资料来源。
- build_system_prompt(identity, campus_docs)：将角色设定、校园资料全文、安全规则拼接为完整的system prompt。
### 4.2 API调用（第108-161行）
chat_with_ai_stream()函数负责与硅基流动API通信：
- 通过requests.post发送流式对话请求，携带system prompt和用户问题
- 逐行解析SSE数据流，实时将AI回答渲染到Streamlit页面
- 包含完善的异常处理：401密钥失效、429请求限频、超时、断网等场景均有对应提示
### 4.3 会话持久化（第164-201行）
支持多会话管理和本地持久化：
- save_conversations()：将所有会话保存到sessions目录，每个会话一个JSON文件，自动清理已删除会话的残留文件
- load_conversations()：启动时从sessions目录加载所有历史会话
- delete_conversation()：删除指定会话的JSON文件
### 4.4 身份分流与推荐提问（第450-519行）
根据用户身份动态渲染不同的推荐提问区域：
- 新生：显示"新生入学"、"校园生活"、"校园交通"三个分类，每个分类下4个推荐问题
- 在校生：显示"校园生活"、"校园交通"两个分类
- 教师：直接显示4个平铺的快捷提问按钮（行政联系、应急处理等）
### 4.5 校园电话黄页（第216-286行）
独立的全屏电话黄页页面，按类别展示校园各部门联系电话：
- 行政服务（校值班室、招生办、教务处等）
- 院系联系（各学院办公室电话）
- 生活服务（后勤、宿管、食堂、校园卡）
- 安全应急（保卫处、校医院）
- 心理关怀（心理咨询中心和热线）
## 五、交互流程
1. 用户打开网页，系统自动加载历史会话或创建默认会话
2. 左侧边栏提供会话管理功能：新建、切换、重命名、删除会话，以及电话黄页入口
3. 右侧主区域首先要求用户选择身份（新生/在校生/教师）
4. 选择身份后，系统根据身份显示对应的推荐提问按钮
5. 用户可点击推荐问题或在输入框手动输入问题
6. 系统加载校园知识库文档，拼接完整Prompt，调用API获取AI回答
7. 回答以流式方式实时显示在页面上，同时记录接口耗时和回复字数
8. 对话记录自动保存到本地JSON文件，下次打开自动恢复
## 六、扩展功能
- 多会话管理：支持同时维护多个独立对话，互不干扰
- 会话重命名：支持自定义会话名称，内置重名检测
- 流式输出：AI回答逐字显示，附带打字光标效果
- 用时字数统计：每次回答下方显示接口耗时和回复字数
- 加载动画：AI回答时显示spinner加载提示
- 全局异常捕获：页面底部try-except包裹主程序，异常时友好提示
- 电话黄页：独立全屏页面，分类展示校园联系方式
## 七、启动方式
在项目目录下执行：
    python -m streamlit run day6_new.py
注意：Windows环境下如遇到"应用程序控制策略已阻止"错误，需使用python -m方式启动，不要直接使用streamlit run命令。
## 八、配置说明
- API密钥：代码第22行API_KEY变量，需替换为有效的硅基流动API密钥
- 知识库路径：代码第25行DATA_DIR变量，指向上级目录的Data文件夹
- 会话存储：代码第28行SESSIONS_DIR变量，指向当前目录下的sessions文件夹
